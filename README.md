# 🐾 AI-Администратор ветеринарной клиники "Хвостики"

Полноценный голосовой ИИ-ассистент для автоматизации работы ветеринарной клиники. Проект построен на базе современной микросервисной архитектуры с разделением фронтенда и бэкенда. Благодаря использованию протокола WebRTC через платформу LiveKit и нативной обработке аудио в Google Gemini Live API, система обеспечивает ультранизкую задержку (Ultra-Low Latency) в диалоге, создавая ощущение общения с реальным человеком.

---

##  Ключевые возможности проекта

* **Ультранизкая задержка (Ultra-Low Latency):** Прямая передача аудиопотока через WebRTC без промежуточных серверов-посредников, пережимающих звук.

* **Нативный голосовой ИИ:** Полная интеграция с Google Gemini Live API. Модель принимает аудио напрямую и генерирует аудиоответ без промежуточного перевода в текст (STT/TTS), что минимизирует задержки и сохраняет интонации.

* **Профессиональный UX/UI маскировки задержек:**

  * Интегрирован кастомный компонент `CallRingtone`, который автоматически воспроизводит телефонные гудки во время подключения клиента к комнате и фазы "разогрева" нейросети (AEC warmup).

  * Как только ИИ-ассистент готов слушать или говорить, гудки бесшовно прекращаются. Для пользователя это выглядит как привычное и естественное ожидание ответа оператора на линии.

* **Динамические одноразовые комнаты:** Логика авторизации автоматически генерирует уникальное имя комнаты на основе временных меток (`Date.now()`) при каждом новом вызове. Это полностью устраняет конфликты сессий, позволяя проводить неограниченное количество повторных звонков на этапе тестирования без перезапуска сервисов.

* **Микросервисная архитектура:** Проект полностью контейнеризирован с помощью Docker и Docker Compose. Компоненты изолированы друг от друга, имеют собственные чистые среды выполнения и управляются единым дирижером.

---

##  Архитектура системы и структура папок

Проект разделен на два независимых слоя:

1. **Фронтенд (`web/`)** — Приложение на Next.js (Node.js), отвечающее за пользовательский интерфейс, визуализацию прыгающего эквалайзера (`BarVisualizer`), управление микрофоном и генерацию JWT-токенов для безопасного подключения клиентов к LiveKit.

2. **Бэкенд-Агент (`agent/`)** — Сервис на Python 3.12, который выступает в роли "воркера" (LiveKit Agent). Он слушает появление новых комнат, автоматически подключается к ним и связывает аудиопоток пользователя с Gemini Live API.

##  Быстрый старт через Docker Compose

Для запуска всей экосистемы одной командой вам потребуется установленный Docker Desktop.

1. **Подготовка конфигурационных файлов**
   
   Перед сборкой необходимо настроить переменные окружения. Создайте два файла конфигурации.

   * **Файл 1:** Создайте `.env` в папке `agent/`:
   
     ```env
     
     LIVEKIT_URL=wss://ваш-проект.livekit.cloud
     
     LIVEKIT_API_KEY=ваш_api_key_livekit
     
     LIVEKIT_API_SECRET=ваш_api_secret_livekit
     
     GOOGLE_API_KEY=ваш_api_key_gemini
     
     ```

   * **Файл 2:** Создайте `.env.local` в папке `web/`:
   
     ```env
     
     LIVEKIT_API_KEY=ваш_api_key_livekit
     
     LIVEKIT_API_SECRET=ваш_api_secret_livekit
     
     NEXT_PUBLIC_LIVEKIT_URL=wss://ваш-проект.livekit.cloud
     
     ```

2. **Сборка и запуск контейнеров**
   
   Откройте терминал в корневой папке проекта (`E:/vet-voice-ai`) и выполните команду:
   
   ```bash
   
   docker-compose up --build
   
   
## Локальная разработка (без Docker)

Если вы хотите развивать проект, менять системные промты Анны или стили интерфейса в режиме реального времени:

1. **Запуск серверной части (Фронтенд)**

cd web

npm install

npm run dev

2. **Запуск ИИ-агента (Бэкенд)

Откройте второе окно терминала и выполните:

cd agent

python -m venv .venv

# Для Windows:
.venv\Scripts\activate

pip install -r requirements.txt

python agent.py dev
